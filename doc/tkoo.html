Global Procedures

tkoo::class

tkoo::class class body

The tkoo::class procedure creates a new class named class, class must not exist. Upon creation of class body will be evaluated in the tkoo::define namespace. The following procedures are defined in the tkoo::define namespace, and are identical to oo::define unless otherwise noted and do not work outside of the class definition.

bind sequence body

This procedure is a shortcut to the tk bind command. Key bindings created by this command will effect all widgets created by the class. Bindings are inherited from parent classes.

constructor argList bodyScript

Procedures from the tkoo::Helpers are imported in the class constructor.

deletemethod name ?name …?

destructor bodyScript

export name ?name ..?

filter ?-slotOperation? ?methodName?

forward name cmdName ?arg …?

method name argList bodyScript

mixin ?-slotOperation? ?className …?

renamemethod fromName toName

self subcommand arg …

superclass ?-slotOperation? ?className?

All widget classes must superclass tkoo::widget.

unexport name ?name …?

variable ?-slotOperation? ?name?

The following variables are added to the variable list:

widCmd contains the original command used to access the widget

pathname contains the pathname of the widget

options contains information about widget options

exists used to determine if the widget already exists



tkoo::wrap

tkoo::wrap original ?new?

This procedure creates classes for Tk widgets, original is the Tk widget to be wrapped. If new is specifed it states the name of the class.



Helper Procedures

These procedures are designed to be used in widget class methods. They are defined in the tkoo::Helpers namespace.

ClassList

Returns a list of all classes that the widget inherits.

Exists widget

Determines if the widget was already created, used in the constructor.

mc src-string ?arg …?

Returns a translation of src-string from msgcat. mc will search classes the object inherits, then search the namespaces of those classes, and the parents of those namespaces, and so on. If no translation string exists, ::msgcat::mcunknown is called and the string returned fom ::msgcat::mcunknown is returned.

mcmax ?src-string …?

Returns the length of the longest translated string.

Option method ?arg …?

The Option command is used to modify and query options for the widget. Option supports the following methods.

add option dbName dbClass default body

The add method adds an option to the widget named option. dbName and dbClass will be used for the option database if support is implemented, until then they provide information supplied by .widget configure. default specifies the value for option if it is not set. body is evaluated anytime the value of an option is modified. option must start with a “-”.

eval option

The eval method is used by the configure method of the widget when an option is changed.

get ?option?

If option is specified, this method will return info about the option in the form of a list consisting of option dbName dbClass default value. If option is not specified, this method will return a list of all options. This method works the same as a widget configure method.

remove option

The remove method removes option from the widget.

set option value

The set method assigns value to option, then evaluates the options body. Upon error, the original value is restored and an error is thrown.

lookup option

Used to find an option from the shortened form.



tkoo::widget class

The tkoo::widget class is the class that all other widget classes are made of, therefore they inherit all of its methods. By default the tkoo::widget class creates a blank frame, this can be overridden in the Create method of the tkoo::widget class. The tkoo::widget class is made of the following methods:

constructor wid ?option value …?

The constructor calls a series of methods to create and initialize the widget. In the event the constructor is overridden the Exists procedure needs to be called first to make sure that a widget with the same name hasn’t already been created. The constructor first creates the variables, then calls the following methods in order CreateOptions, Create, CreateBindings. Finally it renames the widget command and the class.

destructor

The destructor removes the widget and its command.

BindDestroy

The BindDestroy method is called when the widget is destroyed, it destroys the class.

cget option

This method is the same as the widget cget method.

configure ?option value? ...

This method is the same as the widget configure method.

Create wid ?option value …?

This method is called to create the widget. wid is the widget name.

CreateOptions

This method is called to create options for the widget.

CreateBindings

This method handles the bindtags for the widget.

destroy

The destroy method has been unexported to prevent unintentional destruction of widgets. It cannot be used for anything other than destroying the widget.

tkoo::widgets

The following classes are part of the tkoo::widgets package, each one has its own package name as well. These classes can be used like any tk command used to create a widget and inherit all the methods of their Tk counterpart. They all use the tkoo::mixin::help class, for some this is the only distinction between the class and the Tk equivalent widget..
